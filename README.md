# 🎮 Game Server Simulation – Pro Edition

**محاكاة سيرفر ألعاب متعدد اللاعبين**  
مشروع متكامل لمادة **"نظم التشغيل 2" (Operating Systems 2)**  
باستخدام لغة Python وتقنيات *Multithreading* والمزامنة.

---

## 📌 عن المشروع

هذا المشروع عبارة عن **محاكاة تفاعلية لسيرفر ألعاب متعددة اللاعبين**، صُمم خصيصاً لتطبيق المفاهيم الأساسية والمتقدمة لنظم التشغيل، والتي تشمل:

* **الخيوط (Threads) والمعالجة المتعددة (Multithreading).**
* **المزامنة (Synchronization)** باستخدام `Locks` و `Semaphores`.
* محاكاة **حالة التضارب (Race Condition)** وتأثيرها المباشر على البيانات المشتركة.
* تطبيق نمط **المنتج-المستهلك (Producer-Consumer)** عبر قوائم الانتظار.
* **جدولة الأولويات (Priority Scheduling)** لمعالجة الطلبات.
* **إدارة الموارد المشتركة** (CPU, Memory, Network, DB, Game Instances).
* **كشف ومنع الجمود (Deadlock Detection & Prevention)** عبر خوارزمية كشف الدورات (Cycle Detection).
* **مراقبة الأداء (Performance Monitoring)** لحساب الإنتاجية وزمن الاستجابة.

تم بناء المشروع بواجهة رسومية احترافية باستخدام **CustomTkinter** و **Matplotlib**، ويتضمن 10 تبويبات تغطي جميع الجوانب التشغيلية والتحليلية للسيرفر.

## 🧰 المتطلبات الأساسية

تأكد من توفر بيئة العمل التالية قبل التشغيل:
* **Python 3.7+**
* المكتبات الخارجية المطلوبة (موجودة في ملف `requirements.txt`):
  * `customtkinter`
  * `matplotlib`
* مكتبات مدمجة في بايثون: `threading`, `queue`, `random`, `time`, `logging`.

## ⚙️ طريقة التثبيت والتشغيل

**1. استنساخ أو تحميل المشروع**  
تأكد من وجود الملفات التالية معاً في نفس المجلد الأساسي (`game_server/`):
* `game_server_core.py`
* `game_client_sim.py`
* `game_gui.py`
* `requirements.txt`

**2. تثبيت المكتبات المطلوبة**  
افتح الطرفية (Terminal) في مسار المشروع ونفذ الأمر التالي:
```bash
pip install -r requirements.txt

```

*(أو يمكنك تثبيتها يدوياً عبر: `pip install customtkinter matplotlib`)*

**3. تشغيل المشروع**

لبدء المحاكاة وفتح الواجهة الرسومية، نفذ الأمر:

```bash
python game_gui.py

```

## 🖥️ شرح الواجهات (التبويبات)

| التبويب | الوصف والوظيفة |
| --- | --- |
| **1️⃣ Control** | التحكم الرئيسي (تشغيل/إيقاف/إعادة ضبط)، إضافة لاعبين يدوياً أو عبر محاكاة التضارب العشوائية، وإحصائيات سريعة. |
| **2️⃣ Dashboard** | لوحة تحكم شاملة تحتوي على بطاقات أداء (KPIs) ورسوم بيانية حية لتطور أعداد اللاعبين والألعاب. |
| **3️⃣ Players** | جدول ديناميكي للاعبين المسجلين (المعرف، الاسم، السلوك، الحالة، رقم اللعبة)، مع إمكانية طرد أي لاعب. |
| **4️⃣ Games** | عرض جلسات الألعاب النشطة وتفاصيلها (المعرف، الاسم، عدد اللاعبين، المدة)، مع خيار إنهاء اللعبة يدوياً. |
| **5️⃣ Resources** | مراقبة حية للموارد الخمسة الأساسية مع أشرطة تقدم ورسوم بيانية تاريخية لنسبة توفرها. |
| **6️⃣ Race Cond.** | *مدمج في تبويب Control* – رسالة توضيحية توجه المستخدم لأدوات محاكاة التضارب في الواجهة الرئيسية. |
| **7️⃣ Scheduling** | استعراض طابور الأولويات (0-10) للطلبات المنتظرة، مدعوماً بأشرطة تقدم توضح حجم الضغط في كل مستوى. |
| **8️⃣ Deadlock** | أدوات متخصصة لمحاكاة حالة الجمود (Deadlock) واكتشافها فورياً باستخدام خوارزمية DFS مع عرض تقرير الحالة. |
| **9️⃣ Statistics** | تحليلات دقيقة (متوسط زمن الاستجابة، الإنتاجية، إحصائيات الماتشميكينغ، ونسب النجاح) مدعومة بالرسوم البيانية. |
| **🔟 Logs** | سجل الأحداث الشامل مع طوابع زمنية دقيقة، بالإضافة لأزرار لمسح السجل أو تصديره إلى ملف نصي خارجي. |

## 🧪 سيناريوهات الاستخدام والتجربة

### 🟢 1. التشغيل العادي (بدون تضارب)

* اضغط على **Start Server**.
* في قسم "Number of Players"، أدخل عدداً (مثلاً `8`) ثم اضغط **Add Manually**.
* ستلاحظ إنشاء غرفتين تلقائياً (سعة الغرفة 4 لاعبين) وتوثيق ذلك في الـ Logs.

### 🔴 2. محاكاة التضارب (Race Condition)

من خلال قسم "Random Race Simulation" في تبويب **Control**:

* **زر Without Lock:** يضيف عدداً عشوائياً (10-20) **بدون مزامنة**. ستلاحظ حدوث تضارب في التوزيع، طرد بعض اللاعبين (Killed)، وإنشاء غرف أقل من المطلوب.
* **زر With Lock:** يضيف عدداً عشوائياً (2-10) **مع تفعيل القفل**. يتم التوزيع بشكل آمن وصحيح دون فقدان أي بيانات أو لاعبين.

### 📊 3. مراقبة الأداء والموارد

* انتقل إلى **Dashboard** لمتابعة تدفق اللاعبين والألعاب عبر الرسوم البيانية.
* راقب تبويب **Resources** لتحليل استهلاك الموارد لحظة بلحظة.

### 🛑 4. تجربة الجمود (Deadlock)

* من تبويب **Deadlock**، اضغط **Simulate Deadlock** لحقن حالة توقف تام مصطنعة.
* اضغط **Detect Deadlock Now** لتشغيل خوارزمية الكشف ورؤية كيف يقوم النظام بحل المشكلة تلقائياً.

### 💾 5. إدارة السجلات (Logs)

* يمكنك من تبويب **Logs** مراجعة جميع الحركات، ومسح السجل، أو **تصديره إلى ملف `.txt**` للتوثيق والمراجعة.

## 📈 التقنيات والمفاهيم المطبقة بوضوح

| المفهوم (Concept) | مكان وتفاصيل التطبيق |
| --- | --- |
| **Multithreading** | خيوط المستهلكين (3 خيوط مستقلة)، خيوط إدارة اللاعبين، وخيط مراقبة الجمود في الخلفية. |
| **Mutex Locks** | حماية الموارد المشتركة أثناء التعديل، وتجربة الفارق في سيناريو *Race Condition*. |
| **Semaphores** | تحديد والتحكم بالحد الأقصى للمستخدمين المتزامنين لكل مورد. |
| **Producer-Consumer** | طابور الماتشميكينغ (المنتج: يضيف اللاعبين / المستهلك: خيوط المعالجة تسحبهم). |
| **Priority Scheduling** | تصنيف الطلبات الفاشلة مؤقتاً في طوابير حسب الأولوية (من 0 إلى 10). |
| **Deadlock Detection** | تطبيق خوارزمية البحث بالعمق (DFS) لاكتشاف الدورات في مخطط تخصيص الموارد. |
| **Resource Management** | إدارة 5 أنواع مختلفة من الموارد عبر دورات `Acquire` و `Release` الآمنة. |
| **Performance Monitoring** | تتبع وتحليل متوسط زمن الاستجابة والإنتاجية (Throughput) وعرضها رسومياً. |
| **GUI & Visualization** | تصميم واجهة احترافية بـ `CustomTkinter` (Dark Mode)، مدمجة برسم بياني عبر `Matplotlib`. |

## 🎯 أهداف المشروع التعليمية

* استيعاب عملي لآلية عمل **الخيوط (Threads)** وطرق **المزامنة** المتقدمة.
* إثبات أثر غياب الحماية **(Race Condition)** على سلامة البيانات، وكيفية حلها برمجياً.
* تصميم وتطبيق معمارية **المنتج والمستهلك** في بيئة محاكاة قريبة للواقع.
* فهم آليات **الجمود (Deadlocks)**: كيف يحدث، وكيف تكتشفه الخوارزميات، وكيف يتم التعافي منه.
* بناء واجهات رسومية تعرض تدفق البيانات والإحصائيات في الوقت الفعلي (Real-time).
